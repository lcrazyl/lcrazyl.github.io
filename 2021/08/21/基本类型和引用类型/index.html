基本类型和引用类型

发表于 2021-08-21 分类于 JavaScript 阅读次数： | 字数： 749字| 阅读时长: 3 分钟 | Valine：

-

基本类型和引用类型是什么？

  • 基本类型包括：
    • number
    • string
    • boolean
    • undefined
    • null
  • 引用类型：除了基本类型之外的。

    区别：

    存储位置：

  • 基本类型保存在栈中，且进行比较时，比较的时值
  • 引用类型同时保存在栈和堆中，其中栈保存地址，堆保存数据，且进行比较时，比较的是地址

这在说啥呢？又是堆又是栈，请继续看。

什么是栈、堆？

  • 栈：由系统自动分配释放，存储函数的参数值、局部变量等，访问速度块。
  • 堆：由开发人员分配释放，访问速度慢。

    简单来说栈就像租房，可以领包入住，不需要考虑房间的布局、地板的颜色；而堆就像自己动手装修，要考虑房间布局、整体色调。

答案是false,因为引用类型将地址保存在栈中，显然二者的地址是不同的，如下图：

保存方式：

  • 引用类型：当声明一个对象时，会将该对象的地址保存在栈内存；当给变量赋值时，会在堆内存开辟一个空间，并把值保存在堆内存，与此同时，栈内存里的地址指向堆内存里的值。
  • 基本类型：当声明变量时，会在栈内存开辟一个空间将变量值保存在栈内。

浅拷贝：

当对某个数据进行拷贝时，生成的新数据会和原数据相关联，也就是修改原数据或新数据都会相互影响。

浅拷贝只是复制值。

深拷贝：

深拷贝是拷贝一个独立的对象，不仅会拷贝数据，还会拷贝地址。

 let obj = {
        name: "harry",
        age: 20,
        sleep: {
          after: "one",
          evening: "two",
        },
      };
    function deepCopy(target) {
          let newObj = {};
          for (item in target) {
            if (typeof target[item] === "object" && target[item] != null) {
              newObj[item] = deepCopy(target[item]);
            } else {
              newObj[item] = target[item];
            }
          }
          return newObj;
        }

        console.log(obj);//after:"11"
        let newObj = deepCopy(obj);
        obj.sleep.after = "11";
        console.log(newObj);//after:"one"
